# Component Documentation

Per-component technical specifications and behaviour for each module in the Lumina stack.

## Components

| Component | Description |
|---|---|
| `shadow-system.md` | Private settlement layer — note lifecycle, ledger state, Merkle anchoring |
| `compliance-gate.md` | Sanctions screening, jurisdiction rules, KYC tier logic |
| `mpc-policy-engine.md` | Spend ceilings, multi-sig policies, venue allowlists |
| `liquidity-hunter.md` | Cross-venue liquidity scanning and routing optimisation |
| `execution-pods.md` | Modular venue integrations — wallet isolation, execution logic |
| `bridge-adapter.md` | Cross-chain fund finalisation and settlement adapters |
| `vault.md` | Deposit handling — on-chain to private note conversion |

> Documentation is being progressively published. Check back as development progresses.
